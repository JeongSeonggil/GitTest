REST-API URL 설계를 위한 규칙

REST-API

REST API란?

REST(Representational State Transfer)의 약자로
자원을 이름으로 구분하여 해당 자원의 상태를 주고받는 모든 것을 의미

REST API의 작동 방식

REST API는 HTTP 요청을 통해 통신함으로써 리소스 내에서
레코드의 쓰기, 읽기, 수정 삭제 등의 기능을 수행

REST-API URL 규칙

1. URL 마지막에는 슬래시를 포함하지 않는다.

후행 슬래시 (/)는 의미 없지만 혼동을 일으킬 수 있음.

ex.)

http://api.canvas.com/shapes (o)

http://api.canvas.com/shapes/ (x)

2. 계층적 관계를 표시할 때는 구분 기호로 (/)를 사용

슬래시는 리소스 간의 계층적 관계를 나타내기 위해 URL 경로에 사용합니다.

ex.)

http://api.canvas.com/shapes/polygons/squares

3. 밑줄 (_)은 사용 X, 가독성을 높이기 위한 하이픈(-)을 사용

공백이나 하이픈이 사용되는 URL 모든 부분에 사용
여러가지 이유로 밑줄이 가려지거나 숨겨질 수 있음

ex.)

http://api.example.com/blogs/guy-levin/posts/this-is-my-first-post

4. URL 경로에서는 소문자를 사용

대문자 사용으로 인해 문제가 발생할 수 있으므로 소문자를 사용
※ RFC 3986은 체계 및 호스트 구성 요소를 제외하고 URI를 대소문자를 구분하는 것으로 정의

5. 파일 확장자는 URL에 포함 X

대신 Content-Type 헤더를 통해 전달되는
미디어 유형에 의존하여 본문의 콘텐츠를 처리하는 방법을 결정

ex.)

http://api.college.com/students/3248234/courses/2005/fall.json (X)

http://api.college.com/students/3248234/courses/2005/fall (O)

RDB(Relational Database)에서 M:N 관계 관리 (Spring)

M:N 관계 해소 및 활용

M:N 관계란?

M:N 두 개의 테이블이 서로의 행에 대해서 여러개로 연관 되어 있는 상태를
다대다(M:N)관계라고 합니다.

M:N 관계가 생기는 상황

ex.) 개발자가 여러개의 프로젝트를 관리할 때

개발자는 여러 개의 프로젝트를 관리하고, 프로젝트에는 한 명의 개발자가 아닌 여러명의 개발자가 진행할 때
M:N 관계가 발생


※ 1:N의 관계가 아닌 M:N 관계가 발생함.

M:N 관계 해소하기

Relation Table을 따로 생성하여 M:N관계를 해소

project A, B, C와 개발자 D1, D2, D3가 있을 때


Relation Table 생성

M:N 관계 테이블 활용

INSERT, SELECT, DELETE

M:N 관계 테이블에서 생성하기 (INSERT)

ex.) 개발자 D1이 A 프로젝트를 관리하게 되는 경우

result : 0 번째 값인 D1 | A 생성

M:N 관계 테이블에서 조회하기 (SELECT)

ex.) 프로젝트 A를 관리하고 있는 개발자를 조회하는 경우

result : D1, D2 조회

M:N관계 테이블에서 삭제하기 (DELETE)

ex.) 프로젝트 B를 관리하던 D3가 관리를 종료할 경우
